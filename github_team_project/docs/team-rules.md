# 📌 Team Rules & Convention

## 🔹 브랜치 규칙

- main 브랜치 직접 push ❌
- 모든 작업은 feature 브랜치에서 진행

### 🔹 브랜치 네이밍 규칙

작업/#이슈번호-내용

#### ✔ 예시

feature/4-member-a-profile <br>
feature/6-topic-restaurant <br>
feature/8-readme-history-table <br>

---

## 🔹 이슈 & PR 규칙

#### ✔ 예시

#4 member-a 소개 페이지 작성

---

## 🔹 커밋 메시지 규칙

#이슈번호 / 타입: 작업 설명

---

## ✔ 타입 종류

- feat: 기능 추가
- fix: 버그 수정
- docs: 문서 수정
- style: 코드 스타일 변경 (기능 변화 없음)
- refactor: 리팩토링
- test: 테스트 코드
- chore: 기타 작업

#### ✔ 예시

#4 feat: member-a 소개 페이지 작성
#6 fix: 레스토랑 리스트 오류 수정

---

## 🔹 협업 규칙

- 모든 작업은 Issue 기반으로 진행
- PR 생성 후 최소 1명 리뷰 필수
- 충돌 발생 시 작업자가 1차 해결

---

## 📄 PR 템플릿

### 🧾 작업 이슈

관련 이슈: #번호

### ✨ 변경 내용

- member-a 소개 문서 작성
- README 링크 추가

### ✅ 확인 포인트

- 커밋 메시지 규칙 준수 여부
- README 링크 정상 동작 여부
- main 브랜치와 충돌 여부
